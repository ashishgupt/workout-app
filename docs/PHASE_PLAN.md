# Phase Plan

**Phase 0 — Research & decisions (current).**
Frameworks for cardio, strength, tracking. Current-state brief.
Locked decisions. Verification: briefs reviewed and merged.

**Phase 1 — Plan + execution.**
JSON schema for plans and exercises. Static HTML app shows today's
session, audio coach for cardio intervals, fast set logging, fast
pain logging. Verification: two weeks of actual use without
falling back to IntervalTimer or Gym Day. Acceptance criterion:
no in-gym flow > 30 seconds.

**Phase 2 — Tracking + history.**
Logged sessions accumulate. Per-exercise history. Trend views.
Verification: "what did I do last leg day" answerable in <10s.

**Phase 3 — OKRs + health markers.**
Dashboard for habit and health metrics from the OKR framework.
Manual entry + Apple Health export where applicable. Verification:
monthly review takes <15 minutes, produces an actionable insight.

**Phase 4 — Plan revision workflow.**
When framework or current state changes, plan updates without code
changes. Probably: structured prompt → JSON → validated → swapped in.

**Phase 5 — Deferred.**
Adaptive video, Apple Watch integration. Only if Phases 1–4 are
getting daily use.
